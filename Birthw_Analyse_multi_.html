- Régression logistique binaire BIRTHWT¶

  1. Charger le jeu de données 'birthwt' du package 'MASS' dans R.
  2. Étude descriptive des différentes variables de 'birthwt'.
  3. Diviser la base de données en 90% pour l'apprentissage et 10% pour la validation.
  4. Tester l'association de la variable 'low' avec les autres variables par régression logistique binaire.
  5. Évaluer les indicateurs de performance de prédiction du modèle choisi.
  6. Transformer les variables 'ftv' et 'ptl' en variables qualitatives.
  7. Diviser la nouvelle base de données après le codage des variables 'ftv' et 'ptl'.
  8. Évaluer les indicateurs de performance du modèle choisi avec le package 'blorr'.

Présentation des donnèes birthwt :¶

  • low : Variable binaire (0 ou 1) indiquant si le poids du bébé à la naissance est faible (1) ou non (0).

  • age : Âge de la mère en années.

  • lwt : Poids de la mère au dernier cycle menstruel avant la grossesse (en livres).

  • race : Catégorie raciale de la mère (représentée par des entiers, par exemple 1 pour une race, 2 pour une autre, etc.).

  • smoke : Variable binaire indiquant si la mère fumait (1) ou non (0) pendant la grossesse.

  • ptl : Nombre de grossesses antérieures avec un accouchement prématuré.

  • ht : Variable binaire indiquant si la mère a des antécédents d'hypertension (1) ou non (0).

  • ui : Variable binaire indiquant la présence (1) ou l'absence (0) d'irritabilité utérine.

  • ftv : Nombre de visites chez le médecin au premier trimestre de la grossesse.

  • bwt : Poids du bébé à la naissance (en grammes).

Chaque variable représente un facteur potentiel influençant le poids du bébé à la naissance, utile pour l'analyse statistique et la modélisation.

4. Régression logistique binaire¶

La régression logistique binaire repose sur le modèle logistique, qui utilise la fonction de lien logit pour transformer les probabilités en une relation linéaire avec les prédicteurs. Le modèle est formulé comme suit :

  • log(p / (1 - p)) = β0 + β1X1 + β2X2 + ... + βn*Xn

où :

  • p représente la probabilité que la variable dépendante soit égale à 1 (classe positive).
  • β0 est l'intercept du modèle.
  • β1, β2, ..., βn sont les coefficients associés aux variables indépendantes X1, X2, ..., Xn. La fonction logit transforme le rapport des probabilités en une relation linéaire.
  • La variable "y" étant qualitative donc utilisation la fonction "glm" avec une famille binomiale et une fonction de lien logit pour modéliser la relation entre "y" et les autres variables explicatives.
  • Les modèles "glm" sont adaptés pour traiter des variables de réponse catégorielles, et la famille binomiale avec une fonction de lien logit est couramment utilisée pour les variables binaires (0/1).
  • Cette approche permet de modéliser de manière flexible les relations complexes entre les variables, ce qui peut être crucial pour capturer des tendances non linéaires ou des interactions significatives dans vos données.

Tracer une courbe ROC¶

Spécificité et sensibilité :

  • la sensibilité est : TP / (TP + FN) = TP / P.
  • la spécificité est : TN / (TN + FP) = TN / N.

avec

  • TP (true positives) : les prédits positifs qui le sont vraiment.
  • FP (false positives) : les prédits positifs qui sont en fait négatifs.
  • TN (true negatives) : les prédits négatifs qui le sont vraiment.
  • FN (false negatives) : les prédits négatifs qui sont en fait positifs.
  • P (positives) : tous les positifs quelque soit l'état de leur prédiction. P = TP + FN.
  • N (negatives) : tous les négatifs quelque soit l'état de leur prédiction. N = TN + FP.

La courbe ROC (Receiver Operating Characteristic) illustre la performance d'un modèle de classification en affichant

  • la sensibilité par rapport à 1 - spécificité à différents seuils. Typiquement, elle s'étend de (0,0) à (1,1), surpassant la ligne diagonale qui représente une prédiction aléatoire.
  • Plus la courbe s'élève au-dessus de cette diagonale, meilleure est la prédiction.
  • Une courbe idéale toucherait le point (0,1), indiquant une sensibilité parfaite sans faux positifs.
  • L'AUC (Area Under the Curve) quantifie cette performance :
    • 1 indique une prédiction parfaite
    • 0,5 équivaut à une prédiction aléatoire.

## Interpretation

Si un seuil de coupure (cutoff) de 0.2 pour notre modèle , cela signifie que toutes les probabilités prédites supérieures ou égales à 0.2 seront classées comme positives (par exemple, la présence d'une condition ou d'un événement), tandis que celles en dessous de 0.2 seront classées comme négatives

  • Sensibilité (True Positive Rate) : Elle va probablement augmenter parce que le modèle classera plus de cas comme positifs. Cela signifie que le modèle sera plus enclin à détecter les vrais positifs, mais au risque d'augmenter le nombre de faux positifs.

  • Spécificité (False Positive Rate) : Elle va probablement diminuer, puisqu'un seuil plus bas signifie que le modèle est moins strict pour prédire une classe positive. Cela pourrait conduire à une augmentation des cas où les vrais négatifs sont incorrectement classés comme positifs.

  • Valeur Prédictive Positive (Positive Predictive Value) : Elle pourrait diminuer parce que le modèle prédit plus de positifs, y compris de faux positifs, ce qui réduit la proportion de prédictions positives qui sont réellement correctes.

  • Valeur Prédictive Négative (Negative Predictive Value) : Elle pourrait augmenter puisqu'il y aura moins de faux négatifs, mais cela dépend aussi de la prévalence de la condition dans la population.

  • Précision (Accuracy) : Cela dépend de l'équilibre entre vrais positifs et faux positifs. Avec plus de faux positifs possibles, la précision globale pourrait diminuer si le nombre de faux positifs augmente significativement.

En conclusion, abaisser le seuil de coupure à 0.2 va probablement rendre le modèle plus sensible mais moins spécifique. Cela peut être utile dans des situations où manquer un vrai positif est critique (comme dans le dépistage préliminaire d'une maladie grave), mais cela peut également conduire à un grand nombre de faux positifs, qui pourraient entraîner des coûts supplémentaires ou des inquiétudes inutiles. L'évaluation de la performance du modèle avec le package blorr ou tout autre outil statistique devrait fournir des mesures quantitatives de ces effets pour une interprétation plus précise et fondée sur des données.
